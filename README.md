# Club Romance — бот интерактивных новелл для мессенджера MAX

Бот в духе «Клуба Романтики»: истории с ветвлением, характеристиками героини,
отношениями с персонажами, платными сценами за кристаллы и несколькими финалами.
Первая история — **«Наследники Империи»** (2 сезона · 4 главы · 6 основных + 1
секретная концовка).

Платформа — мессенджер **MAX** (от VK), Bot API `https://botapi.max.ru`.

---

## Что внутри

- **Data-driven движок новелл** — истории описываются JSON-структурой, движок
  сам ходит по сценам, считает условия, применяет эффекты, выбирает концовки.
  Новые истории добавляются как JSON, без правки кода.
- **Безопасный вычислитель условий** (`influence >= 2`, `romance_damian > romance_leonard`)
  на разборе AST — без `eval()`.
- **Полное меню** из ТЗ: Истории · Кристаллы (ежедневная награда / подписка /
  рефералы / покупка-заглушка / история операций) · Профиль (достижения,
  мои истории) · Настройки (язык, уведомления, сброс) · Помощь.
- **Кристальная экономика** с журналом операций (`ledger`) и достижениями.
- **Два режима**: polling (разработка) и webhook + FastAPI (продакшн).
- **Оффлайн-симулятор** — прогон истории без живого токена.

## Стек

Python 3.13 · aiohttp · aiosqlite · python-dotenv · (FastAPI + uvicorn для webhook).

---

## Быстрый старт

```bash
# 1. Зависимости
pip install -r requirements.txt

# 2. Конфиг
cp .env.example .env
#   впишите MAX_BOT_TOKEN (из @MasterBot в MAX)

# 3. Запуск (polling — для разработки)
python run.py
```

Откройте бота в MAX, нажмите «Старт» — появится приветствие и главное меню.

### Прод (webhook)

В `.env` задайте `WEBHOOK_BASE_URL` (публичный HTTPS), `WEBHOOK_SECRET`, затем:

```bash
uvicorn webhook_server:app --host 0.0.0.0 --port 8080
# за nginx + certbot; бот сам зарегистрирует подписку BASE_URL/webhook/<secret>
```

> Под нагрузкой используйте только webhook: polling MAX дросселит после ~10 RPS.

## Истории: формат и добавление

Истории лежат в `stories/*.json`. Сцена выглядит так:

```jsonc
{
  "chapter": "Глава 1 — Завещание",     // баннер при смене главы (опц.)
  "title":   "Кабинет адвоката",         // подпись-локация
  "bg":      "lawyer_office",             // ключ фона (картинка, опц.)
  "blocks": [
    { "text": "Реплика рассказчика." },
    { "speaker": "Леонард", "text": "Реплика персонажа." },
    { "speaker": "Героиня", "text": "Видна только при условии",
      "if": "influence >= 2", "fx": { "secret_letter_clue": 1 } }
  ],
  "choices": [
    { "text": "Властный ответ", "fx": { "influence": 1 }, "goto": "scene_x" },
    { "text": "Платная сцена",  "cost": 15, "fx": { "romance_leonard": 2 }, "goto": "scene_y" }
  ],
  "next": "scene_z",                      // авто-переход (кнопка «Дальше»)
  "route_group": "romance",               // развилка по группе концовок
  "complete": { "groups": ["romance","business"], "code_from": "business" },
  "secret": true,                         // открывает секретную концовку
  "final": true                           // терминальный экран
}
```

- **Условия** (`if`): сравнения и `and/or/not` над переменными (`romance_damian > romance_leonard`).
- **Эффекты** (`fx`): числа суммируются, флаги ставятся (`{ "flag": 1 }`).
- **Концовки**: группы `ending_groups` — первый подходящий вариант по порядку.
- **Достижения**: `achievements` — выдаются, когда `if` становится истинным.

Историю «Наследники Империи» удобнее править в **`tools/build_heirs.py`**
(многострочные строки Python, без ручного JSON-эскейпа) и перегенерировать:

```bash
python tools/build_heirs.py          # -> stories/heirs_of_empire.json
```

### Фоны (картинки)

В `backgrounds` истории ключи фонов сейчас `null`. Подставьте публичные URL
картинок — бот прикрепит их к сообщению сцены (image-вложение MAX).

## Тестирование

Оффлайн-прогон истории через настоящий диспетчер с фейковым MAX-клиентом
(токен не нужен):

```bash
python tools/simulate.py            # прогон стратегий + проверки целостности
python tools/simulate.py --print    # + транскрипт одной стратегии
```

Симулятор проверяет: достижимость финалов разными путями, корректность концовок,
секретную концовку и выдачу достижений.

## Заметки по MAX Bot API (грабли учтены в коде)

- Авторизация — заголовок `Authorization: <token>` **без** `Bearer`.
- Форматирование текста — поле `format: "markdown"`, иначе ссылки приходят сырыми.
- `answer_callback` **не** умеет заменить сообщение новой клавиатурой — новый экран
  всегда шлётся отдельным сообщением (так и сделано).
- clipboard-кнопка: копируемый текст в поле `payload`.
- Webhook — отдельная сущность; перевыпуск токена его не сбрасывает. Бот при старте
  снимает чужие подписки (боты «из чужих рук» приходят с webhook на скам-панели).

## База данных

SQLite (файл из `DB_PATH`, по умолчанию `clubromance.db`). Игровые переменные —
JSON-блобом в `progress.vars_json`, поэтому новые истории с новыми переменными не
требуют миграций. На бой при росте нагрузки слой запросов в `app/db.py` можно
заменить на Postgres (как в соседних проектах).

## Кристальная экономика

Старт — 50 💎. Открытие истории — 10 💎. Платные сцены — 15–50 💎.
Заработок: ежедневная награда (+5), подписка (+15, разово), рефералы (+20).
Покупка пакетов — **заглушка** (приём платежей подключается отдельной итерацией).
